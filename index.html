Overview

Executive Summary

Airlines and MRO operators routinely make capital allocation decisions based on maintenance liability data that is never audited at the record level. Cancelled flights get counted as completed cycles. Ghost aircraft accumulate wear with no registry tracking. This project forensically audits 7.07 million BTS flight records against the FAA Master Registry to quantify the actual, verified maintenance liability of the US domestic fleet.

Forensic Analysis

Key Findings

01 — The $17.3M Phantom Liability Catch

$17,336,700

The standard MRL formula applies a $180 per cycle charge to every flight record uniformly. Initial processing revealed 96,315 cancelled flights incorrectly included in the accrual logic — generating phantom maintenance liability that never existed. The pipeline enforces a hard-zero on cancelled flights upstream in the Silver layer before any cost calculation runs.

This is the difference between a dashboard and a forensic audit.

02 — Ghost Aircraft Reconciliation

199 Confirmed

Cross-referencing BTS flight logs against the FAA Master Registry produced 314 null matches — aircraft flying with no official record. Root cause: inconsistent N-prefix formatting across both datasets. After applying tail number standardization: 314 raw nulls → 115 resolved → 199 confirmed Ghost Aircraft. These assets were accumulating flight hours and cycles with zero official registry tracking.

03 — 72.5% of Delays Are Carrier-Controllable

72.5%

The delay decomposition isolates carrier-controllable failures (Carrier Delay) from external factors (weather, NAS, security). 72.5% of all delay minutes are the airline's direct operational responsibility — not weather or ATC. This is the accountability metric that drives intervention decisions and capital reallocation.

04 — Geographic Liability Clustering

TX $1.02B · FL $885M · CA $757M

Maintenance liability concentrates in legacy hub states independently of raw flight volume — pointing to station-level maintenance culture and infrastructure age, not purely operational scale. Generalized fleet upgrades are a misallocation of capital.

Core Pipeline Snippets

MRL Formula — $17.3M Phantom Liability Eliminated

-- MRL Formula: (AirTime / 60 × $250) + $180 per cycle
-- Cancelled flights and null AirTime → hard-zeroed before calculation
-- 96,315 cancelled flights × $180 = $17,336,700 eliminated at source

CASE
    WHEN TRY_CAST(Cancelled AS INTEGER) = 1
      OR TRY_CAST(AirTime   AS INTEGER) IS NULL THEN 0.0
    ELSE ((TRY_CAST(AirTime AS INTEGER) / 60.0) * 250.0) + 180.0
END AS MRL_Liability

Ghost Aircraft Detection — Set Difference

# Set difference: tail numbers flying with NO FAA registry match
# Equivalent to LEFT JOIN WHERE faa.tail IS NULL

active_fleet = set(fact_tails['Tail_Number'].dropna().apply(clean_tail))
registered   = set(merged_master['N-NUMBER_CLEAN'].dropna())

orphans = active_fleet - registered

DAX Analytical Engine

40+ custom DAX measures power the dashboard. Five that define its analytical depth:

Measure Description
Months to Inflection Predicts months remaining before cumulative MRL accrual burns through the $1.5B strategic reserve, based on current 30-day burn rate. Forward-looking, not rear-view.
Scatter Color Logic Dynamic DAX color classification driving the Kill Zone. Evaluates [True Total MRL Liability] > [MRL_Budget_Slicer Value] per aircraft, returning #D92525 (red) or #114477 (navy) on every threshold change.
Controllability Ratio Isolates carrier-controllable delay cost from external (weather, NAS, security) delay cost. Produces the 72.5% accountability figure that drives operational intervention decisions.
Liability Burn Rate
(per flight hour)		 MRL cost accruing per flight hour for any selected aircraft. Surfaces which assets are degrading fastest relative to their utilization — shown live in the drill-through.
Fleet Health % Parametric health score: percentage of registered aircraft currently under the MRL threshold. At $900K threshold: 66.2% — 1 in 3 aircraft in the active fleet is over budget.

How to Run

# 1. Clone the repository
git clone https://github.com/MominPathann/US-Aviation-Reliability-Audit.git
cd US-Aviation-Reliability-Audit

# 2. Install dependencies
pip install -r requirements.txt

# 3. Place FAA registry files in data_raw/
#    Download: https://registry.faa.gov/database/ReleasableAircraft.zip
#    Extract MASTER.txt and ACFTREF.txt into data_raw/

# 4. Run the pipeline
python main_pipeline.py
